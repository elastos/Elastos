    • {% endblock documentation %} {% block serviceicon %} {% endblock serviceicon %} {% block servicetitle %} Upload and Sign {% endblock servicetitle %} {% block description %} The Upload and Sign service is used to upload encrypted file content to Elastos Hive and associate it with a private key Elastos Smartweb Service services. Here is how it currently works:
    • Takes three arguments:
      • api_key: This is the API key that is required. This can be generated by first using Generate API Key Service
      • private_key: This is the private key that will be used to sign the file content. You can generate one by using Create Wallet Service
      • file_to_upload: This is the path of the file to upload to Elastos Hive
    • Elastos Smartweb Service validates whether the passed in API key is valid
    • Elastos Smartweb Service checks whether the user has exceeded their number of API calls for the day. To check the entire list of maximum number of API calls allowed per service, check out the settings file here
    • Encrypts the file content using the passed in private key
    • Uploads the encrypted file content to Elastos Hive
    • Retrieves the IPFS Hash of the file content that was just uploaded and signs that Hash using one's private key
    • Returns the response with all the appropriate info
