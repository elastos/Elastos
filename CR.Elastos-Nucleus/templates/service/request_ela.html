    • {% endblock documentation %} {% block serviceicon %} {% endblock serviceicon %} {% block servicetitle %} Request ELA {% endblock servicetitle %} {% block description %} The Request ELA service is used to request for some test ELA for an address in Mainchain or any of the sidechains Elastos Smartweb Service services. Here is how it currently works:
    • Takes two arguments:
      • api_key: This is the API key that is required. This can be generated by first using Generate API Key Service
      • address: This is the address that you want to request some test ELA for
    • Elastos Smartweb Service validates whether the passed in API key is valid
    • Elastos Smartweb Service checks whether the user has exceeded their number of API calls for the day. To check the entire list of maximum number of API calls allowed per service, check out the settings file here
    • Requests some test ELA for an address. The amount may may vary depending on whether it's a mainchain address or a sidechain address
    • Returns the response with all the appropriate info
