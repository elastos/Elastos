    • {% endblock documentation %} {% block serviceicon %} {% endblock serviceicon %} {% block servicetitle %} Verify and Show Content {% endblock servicetitle %} {% block description %} The Verify and Show service is used to verify file content using a private key and retrieve it Elastos Smartweb Service services. Here is how it currently works:
    • Takes three arguments:
      • api_key: This is the API key that is required. This can be generated by first using Generate API Key Service
      • message_hash: This is the hash of the message that you got when you first used Upload and Sign Service to upload your original file
      • public_key: This is the public key that is generated from your public key. You should have gotten this when you first used Upload and Sign Service to upload your original file
      • signature: This is the signature that is generated after signing file content. You should have gotten this when you first used Upload and Sign Service to upload your original file
      • file_hash: This is the hash of your encrypted file content on Elastos Hive. You should have gotten this when you first used Upload and Sign Service to upload your original file
      • private_key: This is the private key that was used to sign the file content. You should have gotten this when you first used Upload and Sign Service to upload your original file
    • Elastos Smartweb Service validates whether the passed in API key is valid
    • Elastos Smartweb Service checks whether the user has exceeded their number of API calls for the day. To check the entire list of maximum number of API calls allowed per service, check out the settings file here
    • Retrieves the encrypted file content from Elastos Hive
    • Verifies whether the encrypted file content was signed using the provided private key
    • Decrypts the file content using the provided private key
    • Returns the decrypted file content back to the user with all the appropriate info
